# Notas de versão do Scene7 Viewers 4.9.2{#scene-viewers-release-notes}

Aprimoramentos, correções de erros e problemas conhecidos no Scene7 Viewers 4.9.2

## Aprimoramentos nos visualizadores Scene7 4.9.2 {#section-0b0363fae0274737bace119f281e740f}

As atualizações do visualizador são compatíveis com versões anteriores e, portanto, não são necessárias alterações no código da Web existente. No entanto, recomenda-se que você teste os novos visualizadores em nosso ambiente de preparo. Entre em contato com o Suporte Técnico para obter instruções sobre como acessar o servidor de preparo para que você possa testar seus visualizadores. Após isso, você pode verificar seu site para testar as atualizações.

* Requisitos mínimos aprimorados para visualizadores para o iOS6.
* Adição de suporte para rastreamento de eventos personalizado aos visualizadores.
* Adição de suporte para definir a taxa de bits inicial para o Visualizador de vídeo.
* Agora, o padrão do visualizador de vídeo é usar o streaming HLS na área de trabalho do Safari.
* Dicas de ferramentas atualizadas para corrigir vários erros.
* Foram removidas as restrições de recursos sociais nos visualizadores de Vídeo e eCatalog de dispositivos móveis.

## Correções de erros nos Scene7 Viewers 4.9.2 {#section-405e47e202e54943bd843d82415c91cb}

Os seguintes erros foram corrigidos:

* os visualizadores estavam exibindo o tamanho incorreto após retornarem de tela cheia e alternarem ativos.
* os visualizadores exibiam o botão Fechar no modo Tela inteira.
* O visualizador do eCatalog não exibia mapas de imagem no modo retrato em dispositivos móveis.
* O visualizador do eCatalog exibia botões de panorâmica em telefones celulares.
* As dicas de ferramenta padrão do visualizador do eCatalog não se aplicavam se a ID do container não fosse &quot;catálogo&quot;.
* As dicas de ferramentas do visualizador do eCatalog estavam ocultas atrás das miniaturas na visualização da grade.
* O visualizador do eCatalog estava exibindo um divisor de página para uma única página.
* Os mapas de imagem do visualizador do eCatalog não funcionaram no IE9.
* O visualizador de mídia mista redefinia a posição do depurador de vídeo após redimensionar o visualizador.
* O visualizador de mídia mista estava usando a arte incorreta para botões de deslocamento giratório.
* O vídeo do visualizador de mídia mista mostrava dicas de ferramentas em amostras de mídia mista.
* Os botões de rotação do visualizador de mídia mista HTMl5 eram exibidos em tablets.
* As dicas de ferramentas do visualizador de rotação e zoom foram cortadas por borda no visualizador incorporado.
* A posição da dica de ferramenta de Compartilhamento em redes sociais foi deslocada para os botões sociais.
* O formato de dicas de ferramentas de compartilhamento em redes sociais não correspondia às dicas de ferramentas do visualizador.
* As dicas de ferramentas não eram exibidas no modo de tela cheia no Mac OS com o Safari 5.
* O tamanho incorreto do visualizador de vídeo ocorreu para a barra de progresso ao girar o iPad e alternar entre os modos de tela.
* O visualizador de vídeo gerava registros de console por padrão.

## Problemas conhecidos no Scene7 Viewers 4.9.2 {#section-368dd0dc138b44e28b49314e4ad0fa1a}

**Todos os visualizadores Scene7**

* Marcas d&#39;água, ofuscação e bloqueio não são suportadas.

**Todos os visualizadores**

* Os parâmetros com nomes explícitos de instâncias no código devem ser substituídos; os nomes de instâncias em um URL também devem ser substituídos. Por exemplo, `zoomView.iconeffect=0`.
* O recorte do comando do Servidor de Imagens não é suportado.
* O botão Fechar só funcionará se o visualizador estiver aberto em uma janela secundária.
* Para personalizar o formato da dica de ferramenta, adicione `!IMPORTANT` à declaração CSS.

**Visualizador do eCatalog**

* Não há suporte para modelos JavaScript em mapas de imagem.
* Navegar para outra página HTML e retornar ocasionalmente faz com que o visualizador redefina para a primeira página.
* Defina `ImageMapEffect` modificador de sobreposição como `1` para chamar os infiltrados.

* `Frametransition` definido como  `none` ou não  `fade` é suportado.

* Ocasionalmente, o layout da página é exibido incorretamente após a rotação do dispositivo iOS. O zoom na página corrige o layout.

**Visualizador de mídia mista**

* A reprodução da faixa de som não é suportada no momento.

**Visualizador social**

* Para renderizar miniaturas corretamente no email de saída, o modificador `serverurl` deve ter um URL absoluto.

**Visualizador de vídeo**

* A imagem do pôster pode encontrar o erro &quot;tamanho máximo&quot;. O usuário pode precisar aumentar a configuração de limite para o Image Serving Publish.
* As legendas de vídeo exigem um conjunto de regras de empresa se estiverem hospedando uma página HTML fornecida por um servidor externo que não seja um servidor Scene7. Entre em contato com o suporte técnico do Adobe para obter assistência.

**Flash AS3 - todos os visualizadores**

* O duplo codifica o caractere # em nomes de ativos.
* Falha do suporte ao servidor ao carregar animações SWF com vídeos incorporados.
* O suporte ao servidor não carrega as capas do visualizador se compiladas para o Flash Player 6. A solução alternativa é compilar para o Flash Player 7.
* Macintosh OS e Flash Player versão 10,0,32,18: A solução alternativa depende de uma ponte JavaScript em vez de uma conexão local para se comunicar entre a Flex e o Flash. Portanto, o aplicativo Flex deve estar incorporado no invólucro HTML.
* Atualmente, os Visualizadores de Flashes suportam SWFs compatíveis apenas com o Flash 7.
* Devido a um problema com o Java v.1.5.0_06, o componente do servidor não é compatível com essa versão do Java. [http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6274990](http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6274990).
* Para um parâmetro de URL personalizado de capa, o parâmetro no URL de capa deve ser codificado em URL.
* Os conjuntos de imagens com caracteres ASCII estendidos no nome devem ser codificados duas vezes no URL quando enviados para o visualizador ou a solicitação FVCTX falha porque o player do Flash decodifica o nome antes de passá-lo para o código do visualizador.
* Se a página HTML de incorporação tiver conteúdo significativo após o visualizador, é possível que o visualizador chame uma função JavaScript antes que a página seja totalmente carregada. Isso pode interromper o carregamento da página. Uma solução alternativa sugerida é usar um manipulador de eventos onLoad para atrasar a inicialização do visualizador até que o carregamento da página seja concluído.
* As alterações em `S7Config.setFlashParam()` permitem que mais de um parâmetro/valor seja transmitido com esse `syntax S7Config.setFlashParam(<viewer ID>, <parameter name>, <parameter value> [, <parameter name>, <parameter value>])`, mas nesse caso o primeiro parâmetro a ser transmitido deve ser o parâmetro &quot;image&quot; seguido pelo parâmetro &quot;currentFrame&quot; (se necessário) e, em seguida, qualquer outro parâmetro.

* Ao usar o parâmetro `skipFrames=frameList` para omitir imagens, o visualizador retornará um erro se um quadro omitido for chamado diretamente pela função de imagem de alteração do JavaScript, `InitialFrame`, ou no caso de um RenderSet que tenha duas ou mais amostras associadas a uma imagem - quando a combinação de amostras que faz referência ao quadro omitido for selecionada.

* Cortar não é suportado em conjunto com públicos alvos de zoom ou amostras nos argumentos `image=` ou `modifier=`.

* O argumento `rgn` não é suportado para os visualizadores flash para modificadores de imagem.
* Uma nova correção de segurança do IE requer que o usuário ative o Flash e outros aplicativos AtiveX no IE. Consulte o site da Microsoft para obter informações sobre soluções alternativas.
* Problemas de segurança impedem que o visualizador leia a partir de um servidor diferente no mesmo domínio, a menos que ambos os URLs sejam nomes de domínio totalmente qualificados ou que a &quot;política entre domínios&quot; seja definida especificamente para permitir o acesso desse servidor.
* Com os novos recursos de segurança do Flash 7, se os visualizadores tentarem ler a partir de um servidor diferente no mesmo domínio de onde os visualizadores estão localizados (ou seja, eles são redirecionados pelos caminhos em `infoServerUrl`, `searchEngineUrl` ou `serverUrl parameters`), um alerta de segurança notifica o usuário desse fato e pergunta se você deseja permitir isso, a menos que a &quot;política entre domínios&quot; esteja definida para permitir o acesso desse servidor.

   Para obter mais informações sobre como configurar uma &quot;política entre domínios&quot;, consulte o seguinte artigo: [http://kb2.adobe.com/cps/142/tn_14213.html](http://kb2.adobe.com/cps/142/tn_14213.html).

* Erro de exceção gerado quando o visualizador está incorporado à página HTTPS. O problema ocorre devido à comunicação com o registro no servidor. Para obter mais informações: [http://helpx.adobe.com/flash-player/kb/flash-player-issues-secure-sockets.html](http://helpx.adobe.com/flash-player/kb/flash-player-issues-secure-sockets.html)
