## Destaques da versão {#section-524cffc79c1f4bb38627a5ecccd0bf52}

* Suporte adicionado para web design responsivo — fornecimento responsivo de imagens com base em pontos de interrupção e visualizadores responsivos do HTML5 (Conjuntos de imagens, Vídeo, Mídia mista, Catálogo eletrônico, Conjuntos de spam, Conjuntos de amostras, Zoom) que redimensionam dinamicamente com base no layout da página.
* Integração do Adobe Analytics Video Report.
* Personalização avançada do visualizador do HTML5— Crie interações do visualizador personalizado com a página de incorporação da Web.
* Aprimoramento de imagem — Adição da capacidade de recortar imagens em um caminho incorporado na entrega.

## Visualizadores {#section-609885a9c8514a4095da74a0b526138d}

Os itens a seguir foram atualizados:

* O requisito mínimo do sistema para o sistema operacional Windows agora é Windows 7.
* O requisito mínimo do sistema para o Internet Explorer agora é a versão 9.
* O requisito mínimo do sistema para o macOS agora é o macOS 10.8.
* O requisito mínimo do sistema para o Safari agora é a versão 6.

Os seguintes foram refatorados:

* Visualizadores do HTML5 para oferecer suporte a design responsivo.
* Visualizadores de HTML5 para usar marcadores de entrada e tamanho de CSS para controlar a exibição em desktops e dispositivos móveis.

O seguinte agora é compatível:

* Integração de relatórios de vídeo do Adobe Analytics para o visualizador de vídeo HTML5 e o visualizador de mídia mista.
* Rastreamento de eventos de vários visualizadores do HTML5 na mesma página.
* Adicionado `SetIndicator` para HTML5 Zoom Viewer e HTML5 Mixed Media Viewer para representar visualmente amostras em dispositivos móveis.
* Adicionado `getComponent` API para permitir acesso externo aos componentes do visualizador do HTML5.
* Parâmetros de configuração do visualizador do HTML5 passados como um único objeto JSON.
* Legendas da WebVTT a serem passadas como objeto JSON, eliminando a necessidade da configuração do conjunto de regras.
* Modelos JavaScript no Visualizador de catálogo eletrônico do HTML5 para vincular mapas de imagem a páginas externas.
* Vincular a páginas no Visualizador de eCatalog do HTML5.
* `stagesize` para HTML5 Flyout Viewer.
* `initComplete` manipulador de visualizadores do HTML5 para acionar funções após os visualizadores.
* Manipulador do Analytics para visualizadores do HTML5 para oferecer suporte ao rastreamento de terceiros.
* Adição do botão &quot;Selecionar tudo&quot; para facilitar a obtenção do Link e do Código incorporado para compartilhamento social.
* Adição de redimensionamento para FlyoutZoomView.
* Definir pontos de interrupção de recarregamento de imagem para FlyoutZoomView quando usada com design responsivo.
* Adição do suporte a cliques para ajuste do volume.
* SpinView e ZoomView para aplicar gestos à incorporação da página da Web quando não interagir com o visualizador.
* Adicionado `unload()` API para SpinView e ZoomView para permitir a eliminação do componente no tempo de execução.
* O Visualizador de vídeo para aplicar gestos à página da Web de incorporação quando não interagir com o visualizador.
* Definindo a região de zoom inicial para ZoomView.
* ZoomView para ocultar `IconEffect` componente quando o zoom não é possível

Consulte também [Guia de referência de visualizadores](https://experienceleague.adobe.com/docs/dynamic-media-developer-resources/library/homeviewers.html?lang=en).

* Estenda a pop-up de detalhes do trabalho para incluir o tempo para cada registro detalhado.

## Correções de erros {#section-456515477c8a4aa69309de43292ce5ed}

**Visualizadores**

* Possíveis problemas de segurança dos visualizadores de Fastzoom e Flyout do Flash.
* Visualizadores do HTML5 não carregados devido ao UserAgent não reconhecido.
* HTML5 O recurso de arrastar e soltar do mouse do Visualizador de Catálogo Eletrônico do5 não funciona no Internet Explorer com páginas que contêm mapas de imagem.
* HTML5 Video Viewer impede o acesso ao plug-in do Flash.
* Os visualizadores do HTML5 impedem a impressão no navegador Firefox.
* HTML5 O Zoom Viewer (Visualizador de zoom) exibe uma pequena tremulação ao iniciar ou redefinir o zoom.
* Suporte HTTPS para visualizadores HTML5 e AS3.
* autoResize não deve estar no código HTML Flyout viewer Copy URL/Embed .
* Nenhum modificador videoserverurl no URL de cópia do visualizador de vídeo do HTML5.
* O parâmetro `contenturl` é do URL de visualização do Visualizador do HTML5 do servidor de publicação.

**Imagem**

* Não é possível adicionar a exibição ajustada como Imagem padrão na configuração Publicação de imagem.
* Altere o rótulo colorManagementOptions &quot;Convert to SRGB (padrão)&quot; para &quot;Default Color Management&quot;.
* Quando em Arquivo > Editar informações > selecionar vários ativos de imagem, as informações de edição não estavam disponíveis.

**Plataforma**

* Adição de uma mensagem de alerta ao editar a configuração do Adobe Analytics e tentar escapar da configuração sem salvar.
* A referência de opções de projeto é redefinida como Nenhum ao editar um trabalho de upload agendado.
* A pasta File-Move não deve ser mostrada para a função Usuário.
* Não foi possível ver algumas das guias na exibição de Detalhes.
* Erro de edição do Web-to-print quando a versão de depuração do Flashplayer é instalada.
* &quot;Space After&quot; não funciona; deve usar `sa`atualmente é `margb`.
* Ao inicializar as configurações da empresa, o Scene7 Publishing System envia a opção duplicar `<defaultValue>` na API `<createMetadataField>`.

* O plug-in do Flash e o Scene7 Publishing System travaram devido a muitas pastas não terem memória Flash suficiente.
* Ao remover o valor do campo Data/hora definido pelo usuário, o Scene7 Publishing System não envia uma solicitação ao IPS (Image Publishing System).
* Se as opções de codificação de vídeo estiverem ativadas, o reprocessamento de um arquivo PDF ou PSD encontrou a mensagem de aviso de Codificação de vídeo adaptável.
* Falha ao exportar ativo quando o SSL está ativado.
* A senha incorreta estava sendo exibida no email Esqueceu a senha .
* Qualquer solicitação do Administrador da Web do sistema de publicação do Scene7 é redirecionada para HTTP.
* Todas as predefinições do conjunto de lotes estavam sendo aplicadas durante o upload se nenhuma predefinição fosse selecionada.
* O Scene7 Publishing System solicita s7info por meio de HTTP ao carregar conteúdo s7info.
* O URL no email Esqueceu a senha é HTTP, mesmo se ele veio do SSL Scene7 Publishing System.
* Caracteres especiais eram codificados em um email &quot;Esqueceu a senha&quot; formatado para HTML, fazendo com que a senha não fosse exibida corretamente.
* As informações da conta FTP da Scene7 estão ausentes no email de boas-vindas de uma conta de avaliação
* Falha no reprocessamento do trabalho quando a opção do projeto e a opção de corte estão selecionadas.
* O upload do arquivo não funciona quando as opções de trabalho são redefinidas para os padrões da empresa.
* O rótulo Empresa não é exibido completamente na página Configuração do Adobe Analytics.
* Não é possível importar metadados.

## Problemas conhecidos {#section-b2608177928b4174b080d89c938bb0d3}

* A atualização de maio de 2014 do Adobe Illustrator e do Adobe InDesign deve introduzir incompatibilidades com plug-ins que criam o Scene7 FXG. Essas incompatibilidades afetam os usuários da solução Scene7 Web-to-Print. Se você usar o Scene7 Web-to-Print, não atualize o Illustrator e o InDesign para esta atualização do Creative Cloud. Em vez disso, continue usando a versão anterior a maio de 2014. A Adobe Inc espera a restauração da compatibilidade da criação de FXG do Scene7 com versões mais recentes do Illustrator e do InDesign no período do último trimestre de 2014.
* O botão Incorporar código não funciona quando a predefinição do Universal Viewer contém a predefinição AS2 padrão desativada.
* O reprocessamento do Conjunto de vídeos adaptáveis sempre cria um novo Conjunto de vídeos adaptáveis.
* A interface do usuário da caixa de diálogo Detalhes do trabalho é truncada à direita após adicionar Tempo nos detalhes.

## Não há mais suporte para recursos {#section-7b04b6fdb8d1473880ccaf84211254ad}

* A separação de sílabas de texto é incompatível com a versão do último trimestre do Scene7 6.9. Os dicionários de separação de sílabas definem como as palavras podem ser divididas e separadas por sílabas para suportar um fluxo de texto ideal.
* A partir de 31 de janeiro de 2014, a Scene7 encerrou oficialmente o suporte para a plataforma do visualizador DHTML.
* Os banners dinâmicos (Flash) estão sendo descontinuados nesta versão. A adoção desse recurso tem sido baixa, à medida que outras tecnologias surgiram para criar, gerenciar e fornecer banners personalizados em sites. A funcionalidade Banner dinâmico (Flash) do Scene7 continua a funcionar no lado do servidor, mas não é mais suportada.
* Lembrete: O Servidor de metadados e o uso de APIs associadas foram oficialmente incompatíveis a partir da versão anterior do Scene7.
* As extensões Creative Suite do Scene7 (painéis de Flash) foram descontinuadas no final de 2014. Outros desenvolvimentos dos fluxos de trabalho de assimilação de conteúdo estão focados no Adobe Experience Manager Dynamic Media e no Creative Cloud. Para o Scene7, os recursos de upload e exportação no Sistema de publicação do Scene7 permanecem e podem ser usados.
* A partir de maio de 2014, o Akamai não será mais compatível com a notificação por email quando a Invalidação de cache no Scene7 for concluída. O tempo de processamento típico para invalidação de CDN é de aproximadamente oito minutos. Se houver uma fila grande de solicitações ou muitos URLs, o tempo de processamento poderá demorar um pouco mais.
* Categoria &quot;Predefinições universais&quot; no URL e na guia Código incorporado na exibição Detalhes.
* Transmissão RTMP para visualização detalhada do vídeo.
* &quot;Comunicações de email&quot; e o servlet SendMailSimple.jsp do Sistema de publicação da Scene7.
* Seção da conta FTP do Scene7 no modelo de email de boas-vindas para o usuário do Media Portal.
* dc.title do painel de filtro, pois fazia com que a instância do IPS (Image Publishing System) ficasse inativa.
* Recursos sociais exibidos no modo de tela cheia nos visualizadores do HTML5.
* Botão Fechar exibido no modo de tela cheia nos visualizadores do HTML5.
